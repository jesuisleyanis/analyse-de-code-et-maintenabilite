Documentation API v4.0

Multiplayer, Scaling & Automation Edition

Concepts Clés

  • Multi-joueurs : Chaque appel nécessite un playerId unique.
  • Scaling : Le boss gagne 20% de points de vie max à chaque défaite.
  • Automation : L'endpoint /auto-tick permet d'infliger des dégâts sans riposte.

Endpoints

GET /api/v1/hero/{playerId}

Récupère l'état complet du joueur et du monde.

POST /api/v1/battle/attack

Body: { "playerId": "string", "mode": "SOLO|COOP|PVP", "targetId": "string?" }

POST /api/v1/battle/auto-tick

Applique les dégâts du script automatique (Incrémental).

POST /api/v1/hero/use-item

Body: { "playerId": "string", "itemIndex": int }

Évaluation Refactoring
